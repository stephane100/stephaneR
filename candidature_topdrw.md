## 3. Conseils pour le premier échange

### Questions à poser (en visio/appel) pour cadrer la mission

1. **Périmètre fonctionnel** : Quels modules ? (Tâches / Kanban / Gantt / Time-tracking / Facturation / Rapports / Notifications / Multi-utilisateurs avec rôles ?)
2. **Volume utilisateurs** : Combien d'équipes / projets / utilisateurs simultanés visés ?
3. **Cible technique** : Existe-t-il déjà une stack imposée ou je suis libre de proposer ?
4. **Plateforme** : Web only, ou aussi mobile (PWA / app native) ?
5. **Intégrations** : Slack ? Google Calendar ? Notion ? Trello import ? Stripe ?
6. **Délais** : Date de livraison MVP visée ? Budget ?
7. **Hébergement** : Tu as une préférence (OVH / GCP / AWS / Hostinger) ?
8. **Maintenance post-livraison** : Mission ponctuelle ou suivi sur plusieurs mois ?

### Stack que tu peux recommander (efficace + rapide à livrer)

- **Frontend** : Vue 3 + Nuxt 3 + TailwindCSS + Pinia
- **Backend** : Laravel 11 (API REST/Sanctum) ou NestJS si Node préféré
- **DB** : PostgreSQL (mieux que MySQL pour les requêtes analytiques de suivi de projets)
- **Auth** : Laravel Sanctum + multi-rôles (admin / chef de projet / membre / client)
- **Realtime** : Laravel Reverb (websockets natifs) ou Pusher pour les notifs
- **Hébergement** : Hostinger VPS / OVH / DigitalOcean (rapport qualité-prix imbattable à Mada)
- **CI/CD** : GitHub Actions + déploiement auto

Plutôt que de partir de zéro, je peux vous livrer un MVP fonctionnel en 4-6 semaines en m'appuyant sur des briques que je maîtrise (auth, rôles, API, dashboards) et concentrer le temps sur **votre logique métier spécifique**."
